1. INTRODUCTION TO OPERATING SYSTEMS

1.1 Definition

  • Operating System (OS) is a collection of system programs that control the operation of a computer system
  • Acts as an intermediary between the user and computer hardware
  • Provides an environment where users can execute programs conveniently and efficiently
  • The one program running at all times on the computer (called the kernel)

1.2 Primary Goals

Goal Description
Convenience Makes computer more convenient to use
Efficiency Allows computer system resources to be used efficiently
Ability to Evolve Permits effective development, testing, and introduction of new functions without interfering with service

1.3 OS Functions

  1. Resource Allocation - Manages memory, processors, devices, and information
  2. Interface - Acts as interface between user and hardware
  3. Execution Support - Facilitates program creation, modification, and execution
  4. I/O Operations - Handles input/output programming details

2. TYPES OF OPERATING SYSTEMS

2.1 By User Type

Single-user Operating System

  • Designed for one user at a time
  • Examples: Windows 95, Windows XP, Windows 7
  • Single keyboard, mouse, monitor
  • User must finish before another can access

Multi-user Operating System

  • Multiple users access simultaneously via network
  • Examples: UNIX, Linux, Mainframes
  • More complex, manages multiple requests
  • Centralized servers, shared resources

2.2 By Processing Type

Batch Operating System

  • Sequence of jobs without manual intervention
  • Jobs grouped with similar needs
  • Processor alternates between user programs and monitor
  • Advantages: Maximizes processor utilization
  • Disadvantages: No user interaction, slow response time

Time-sharing (Multitasking) Operating System

  • Multiple users share computer simultaneously
  • Fast switching gives illusion of dedicated system
  • CPU divided into time slots
  • Used when users connected via communication networks
  • Examples: UNIX, Linux, Windows NT Server, Windows 2000 Server

Multiprogrammed Batch System

  • Multiple jobs ready for execution (job pool)
  • OS decides which job executes next
  • Maximizes CPU utilization
  • I/O overlap with processor activity
  • CPU never sits idle

Multiprocessor System (Parallel/Tightly-coupled)

  • Two or more processors work together
  • Advantages:
    • Reduced cost (shared resources)
    • Increased reliability (failure doesn't crash system)
    • Increased throughput
  • Types:
    • Symmetric (SMP): All processors equal, I/O on any processor
    • Asymmetric: Master processor controls slaves, I/O on master

Distributed System (Loosely-coupled)

  • Computation distributed among several processors
  • Each processor has own local memory
  • Communicates via network
  • Advantages:
    • Resource sharing
    • Computation speedup
    • Reliability (failure tolerance)

Clustered System

  • Multiple CPUs accomplish single task
  • Shared storage, linked via LAN
  • Types:
    • Asymmetric: One hot standby, one active
    • Symmetric: Both run applications, monitor each other

2.3 Real-time Operating System (RTOS)

Hard Real-time System

  • Guarantees critical tasks complete on time
  • Secondary storage limited or missing
  • Virtual memory absent
  • Examples: Scientific experiments, medical imaging, industrial control, weapons systems

Soft Real-time System

  • Less restrictive, doesn't guarantee completion time
  • Critical task gets priority until completion
  • Examples: Multimedia, virtual reality, undersea exploration, planetary rovers

2.4 By Device Type

Stand-alone Operating System

  • Desktop/notebook computers
  • Examples: DOS, Windows XP/7, UNIX, Linux, Mac OS

Server Operating System

  • Supports network operations
  • Examples: Windows Server 2008, UNIX, Linux, Solaris, NetWare

Embedded Operating System

  • Handheld computers and small devices
  • Resides on ROM chip
  • Examples: Windows Embedded CE, Windows Mobile, Palm OS, Android

3.5 Process Creation

  • Events causing creation:
    • User request
    • System initialization
    • Batch job initialization
    • System call (fork())
  • Parent-child relationship:
    • Child created using fork()
    • Child has one parent, parent may have many children
    • Same memory image, open files, environment
    • Distinct address spaces

3.6 Process Termination

Causes:

  • Natural completion of execution
  • Parent requests termination
  • Resource violation (e.g., write to read-only file)
  • I/O failure
  • Parent termination (cascade)
  • Insufficient memory

4. THREAD MANAGEMENT

4.1 Thread Definition

  • Lightweight Process (LWP)
  • Basic unit of CPU utilization
  • Separate flow of control with own:
    • Program counter
    • System registers
    • Stack
    • Thread ID

4.2 Thread vs Process

Aspect Process Thread
Weight Heavyweight Lightweight
Resources Resource intensive Less resources
Switching OS interaction needed No OS interaction
Memory Separate memory Shared memory
Blocking One blocked blocks all Others can run
Communication Complex (IPC) Simple (shared memory)

4.3 Thread Components

  • Thread ID
  • Program counter
  • Register set
  • Stack

4.4 Shared Thread Resources

  • Code section
  • Data section
  • OS resources

4.5 Thread Types

User-Level Threads

Aspect Description
Management User-managed, kernel unaware
Implementation Thread library at user level
Speed Fast to create and manage
Portability Runs on any OS
Multiprocessing Cannot use multiprocessors
Blocking Entire process blocked

Kernel-Level Threads

Aspect Description
Management OS-managed
Implementation Kernel support
Speed Slower to create/manage
Portability OS-specific
Multiprocessing Can use multiprocessors
Blocking Other threads can run

4.6 Multithreading Models

Many-to-One

  • Multiple user threads â†’ One kernel thread
  • User space management
  • Blocking system call blocks entire process
  • Cannot run parallel on multiprocessors

One-to-One

  • Each user thread â†’ One kernel thread
  • More concurrency than many-to-one
  • Supports parallel execution
  • Used by: Windows NT, OS/2

Many-to-Many

  • Multiple user threads â†’ Multiple kernel threads
  • Best concurrency
  • Developers create many user threads
  • Kernel threads run in parallel
  • Used by: Solaris

4.7 Thread Advantages

  • Minimized context switching time
  • Concurrency within process
  • Efficient communication
  • Economical creation/context switch
  • Better multiprocessor utilization

5. CPU SCHEDULING

5.1 Scheduling Types

Type Description
Long-term Which jobs admitted to system
Short-term Which process gets CPU next
Medium-term Swapping decisions

5.2 Scheduling Algorithms

First-Come-First-Served (FIFO)

  • Simple queue, first job first served
  • Advantages: Simple, fair
  • Disadvantages: Long waiting for short jobs

Round Robin

  • Each process gets time slice (quantum)
  • Preemptive scheduling
  • Advantages: Fair, good response time
  • Disadvantages: Overhead of context switching

Priority Scheduling

  • Processes assigned priorities
  • Highest priority runs first
  • Can be preemptive or non-preemptive
  • Aging: Increase priority of waiting processes

Shortest Job First (SJF)

  • Process with shortest burst time next
  • Optimal for minimum average waiting time
  • Problem: Predicting burst time

Multilevel Queue

  • Different priority queues
  • Each queue may have different scheduling

5.3 Context Switching

  • Definition: Saving/restoring process state during switch
  • Requirements:
    • Timed interrupt (1-10ms)
    • Save interrupted program state
    • Restore next program state
  • Dispatcher Functions:
    • Switch context
    • Switch to user mode
    • Jump to correct program location

5.4 Scheduling Methods Comparison

Method Type Overhead Starvation
FIFO Non-preemptive Low No
Round Robin Preemptive Medium No
Priority Both Low Possible
SJF Non-preemptive Low Possible
Multilevel Both High Possible

5.5 Cooperative vs Preemptive

  • Cooperative: Process voluntarily gives up CPU
    • Problem: One hanging process crashes system
    • Example: Windows 3.1
  • Preemptive: OS forces process to give up CPU
    • Better response, prevents lockups
    • Example: Windows NT

6. MEMORY MANAGEMENT

6.1 Memory Management Activities

  • Track which parts of memory are used and by whom
  • Decide which processes load when memory available
  • Allocate and deallocate memory
  • Swap between memory and disk

6.2 Memory Terms

Term Definition
RAM Random Access Memory - volatile temporary storage
Virtual Memory Disk area used as additional RAM
Swap File Area of disk for virtual memory
Fragmentation Memory broken into small pieces

6.3 Memory Allocation Methods

  • Contiguous Allocation
  • Non-contiguous Allocation
    • Paging
    • Segmentation

6.4 Virtual Memory

  • Technique allowing execution of incomplete programs
  • Part in RAM, part on disk
  • Advantages:
    • Larger programs can run
    • More processes in memory
    • Faster program loading

6.5 Memory Protection

  • Prevents processes from accessing other process memory
  • Uses base and limit registers
  • Ensures OS protection from user programs

7.4 File System Concepts

Root File System

  • Created during OS installation
  • Fixed size table
  • Contains files and directories
  • Entry contains: filename, cluster number, length, type, dates, permissions

Cluster

  • Grouping of sectors
  • Minimum allocation unit
  • Files allocated in cluster increments

File-system Integrity

  • Checked during startup/shutdown
  • Tools: fsck (UNIX), Scandisk/Chkdsk (Windows)

Defragmentation

  • Reorganizes scattered file clusters
  • Makes clusters contiguous
  • Improves access speed

9. DEADLOCKS

9.1 Deadlock Definition

  • Situation where waiting processes never change state
  • Resources held by other waiting processes

9.2 Necessary Conditions (All Must Hold)

Condition Description
Mutual Exclusion At least one resource non-shareable
Hold and Wait Process holds resources while waiting
No Preemption Resources cannot be taken away
Circular Wait Circular chain of processes waiting

9.3 Deadlock Handling Methods

Method Description Complexity
Prevention Ensure one condition can't hold High
Avoidance Banker's algorithm, safe states Medium
Detection Find deadlocks, recover Medium
Ignorance Pretend deadlocks don't occur Low

9.4 Deadlock Recovery

  • Terminate processes (one or all)
  • Preempt resources
  • Rollback to safe state

Turnaround Time

Turnaround Time = Waiting Time + Burst Time

Waiting Time

Waiting Time = Turnaround Time - Burst Time

Average Waiting Time (SJF)

Average Waiting Time = Sum(Waiting Time) / Number of Processes

Response Time

Response Time = Time First Response - Submission Time
