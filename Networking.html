1. Introduction to Computer Networks

What is a Computer Network?

A collection of computers and devices connected together through connecting media to perform tasks such as:

  • Resource Sharing: File sharing, device sharing, software sharing
  • Communication: Voice and video calls
  • Internet Access: Shared internet connectivity

Basic Network Terminology

Term Definition
NIC (Network Interface Card) Hardware that plugs into the motherboard and directly accesses the network
MAC Address Physical address, unique worldwide, burned on the NIC card (48-bit, 6 bytes, hex format)
IP Address Logical address that identifies each device on an IP network
Protocols Communication rules that all entities must agree on

Network Devices Comparison

Device Layer Function
Hub Layer 1 Multiport repeater, broadcasts to all ports
Switch Layer 2 forwards traffic based on MAC address
Router Layer 3 Routes packets between networks based on IP address

3. Network Topologies

Types of Network Topologies

Bus Topology

  • All devices connected to a central cable (bus/backbone)
  • Both ends must be terminated with terminators
  • Advantages: Simple, least cable required, reliable
  • Disadvantages: Faulty cable affects entire LAN, difficult troubleshooting, no security, slow during peak traffic

Star Topology

  • All devices connected to a centralized unit (Hub or Switch)
  • Hubbed Star: Physical star, logically bus (broadcast)
  • Switched Star: Central node acts as frame switch, forwards only to destination
  • Advantages: One PC failure doesn't affect network, easy expansion, centralized management, easy troubleshooting
  • Disadvantages: Central device failure affects entire network

Ring Topology

  • Devices connected in a closed loop
  • Data travels in one direction using a token
  • Advantages: Fair (equal access), performs well under heavy traffic
  • Disadvantages: Network expansion affects operation, one node failure affects entire network, difficult troubleshooting, slow with many PCs

Mesh Topology

  • Each device connected to every other device
  • Cable calculation: CN = (D √ó (D-1)) / 2
  • Types: Full Mesh, Partial Mesh
  • Advantages: Highest fault tolerance, redundancy, secure
  • Disadvantages: Very expensive due to cable requirements

Hybrid Topology

  • Combines bus, star, and ring topologies
  • Advantages: Easy network expansion, flexible
  • Disadvantages: Hub failure affects connections

Topology Selection Factors

  • Type and number of equipment
  • Cost
  • Physical vs Logical topology considerations

4. Network Models

Peer-to-Peer (P2P) Networks

  • No dedicated resources for services
  • All nodes are equal
  • Example: Windows Workgroup
  • Advantages: Easy to work with, no dedicated server needed

Client/Server Networks

  • Server: Dedicated nodes providing services
  • Client: Nodes requesting and using services
  • Examples: Mail Server, Web Server, File Server, Print Server
  • Characteristics: Servers are more powerful, fewer servers than clients

Comparison: P2P vs Client/Server

Feature Peer-to-Peer Client/Server
Cost Lower Higher
Scalability Limited High
Security Lower Higher
Management Decentralized Centralized
Performance Variable Consistent

5. Basic Network Elements

Hardware Components

  • Computers/Peripherals: Devices with NIC (PCs, laptops, printers, cameras, smart TVs, mobile phones)
  • NIC (Network Interface Card): Internal (motherboard) or external (USB, wireless)
  • Connecting Media: Wired (cables) or Wireless
  • Networking Devices: Hubs, Switches, Routers, Repeaters

Software Components

  • Messages: Information traveling over the medium (emails, WhatsApp, etc.)
  • Protocols: Govern message flow (HTTP, HTTPS, FTP, RDP)

Simple Network Requirements

  1. Hardware: Computers, NICs, Connecting Media, Networking Devices
  2. Software: Protocols for communication rules

6. OSI Reference Model

The Seven Layers (Top to Bottom)

Layer Number Function Protocols/Examples
Application 7 Interface to end users, file transfer, network management, email HTTP, FTP, SMTP, POP3, Telnet/SSH, RTP
Presentation 6 Data translation, coding/decoding (EBCDIC to ASCII), common presentation JPEG, ASCII, EBCDIC
Session 7 Establishes, manages, terminates sessions, synchronization with checkpoints NetBIOS, RPC
Transport 4 Organizes data into segments, reliable transport, error recovery, flow control TCP, UDP
Network 3 Organizes data into packets (datagrams), logical addressing, routing IP, ICMP, ARP
Data Link 2 Reliable data transfer across physical link, frames, error control, hop-to-hop addressing Ethernet, MAC addresses
Physical 1 Transmission of bit stream over physical link, mechanical/electrical specifications Cables, connectors, signals

Benefits of Layered Model

  • Reduces complexity
  • Standardizes interfaces
  • Facilitates modular engineering
  • Simplifies teaching and troubleshooting
  • Allows multiple-vendor development
  • Changes in one layer don't affect others

Data Encapsulation Process

Application Data ‚Üí Segment (TCP/UDP) ‚Üí Packet (IP) ‚Üí Frame (MAC) ‚Üí Bits

De-Encapsulation

Reverse process where each layer strips its header and passes data to the upper layer.

7. TCP/IP Protocol Suite

Overview

  • TCP/IP: Transmission Control Protocol/Internet Protocol
  • Open standard protocol, not tied to one vendor
  • Default protocol for modern OS (Windows, UNIX, Linux)
  • Versions: IPv4 (current), IPv6 (next generation)

TCP/IP vs OSI Model Comparison

TCP/IP Layers OSI Layers
Application Layer 5, 6, 7 (Session, Presentation, Application)
Transport Layer 4 (Transport)
Internet Layer 3 (Network)
Network Access Layer 1, 2 (Physical, Data Link)

Advantages of TCP/IP

  • Standard protocol
  • Direct internet access
  • Routable
  • Cross-platform compatible

Disadvantages of TCP/IP

  • Difficulty of setup
  • Slower than IPX/SPX and NetBEUI

Major Networking Protocols

Protocol Vendor Type
TCP/IP Open Standard, Routable
IPX/SPX Novell Not free, Not standard
NetBEUI Microsoft Not routable
AppleTalk Apple Legacy
DLC Various Data Link Control

TCP/IP Architecture Layers

Network Access Layer

  • Physical Layer: Transmission medium, signal rate
  • Data Link Layer: Logical interface, hop-to-hop addressing, error detection

Internet Layer (IP Layer)

  • Protocol: IP (Internet Protocol)
  • Packet: Datagram (Header + Data)
  • Functions: Routing, logical addressing (IPv4/IPv6)

Transport Layer

  • Protocols: TCP, UDP
  • Functions: Process-to-process communication, port numbers

Application Layer

  • Protocols: HTTP, FTP, SMTP, DNS, Telnet, etc.

8. IP Addressing (IPv4)

IP Address Basics

  • Definition: Logical address identifying each device on an IP network
  • Format: 32-bit divided into 4 octets (8 bits each)
  • Notation: Dotted Decimal (e.g., 192.168.1.1)
  • Address Space: 2^32 = 4,294,967,296 addresses

IP Address Classes

Class First Octet Range Default Subnet Mask Purpose
A 1 - 126 255.0.0.0 Large networks
B 128 - 191 255.255.0.0 Medium networks
C 192 - 223 255.255.255.0 Small networks
D 224 - 239 N/A Multicast
E 240 - 255 N/A Experimental

Private IP Addresses (RFC 1918)

Class Private Address Range
A 10.0.0.0 - 10.255.255.255
B 172.16.0.0 - 172.31.255.255
C 192.168.0.0 - 192.168.255.255

Reserved Addresses

Type Address Purpose
Network Address First address in subnet Reserved, used for routing
Broadcast Address Last address in subnet All devices on subnet
Loopback 127.0.0.1 Testing TCP/IP protocol
APIPA 169.254.0.0 - 169.254.255.255 Auto-configuration when DHCP fails

IP Address Assignment Methods

Manual Configuration

  • Set IP address
  • Set Subnet mask
  • Set Default Gateway
  • Set DNS server

Automatic Configuration (DHCP)

  • DHCP Server: Assigns IP configuration dynamically
  • DORA Process: Discover ‚Üí Offer ‚Üí Request ‚Üí Acknowledge
  • Includes: Host IP, Subnet mask, Default Gateway, DNS server, Lease Time

APIPA (Automatic Private IP Addressing)

  • Used when DHCP server is unavailable
  • Auto configuration
  • Allows LAN users to communicate if DHCP fails
  • Not routable

Delivery of IP Packets

Type Description
Direct Delivery Sender and receiver on same network
Indirect Delivery Sender and receiver on different networks (requires router)

NAT (Network Address Translation)

  • Allows a site to use private addresses for internal communication
  • Translates to global Internet addresses for external communication
  • Solves IPv4 address exhaustion

Physical (MAC) Address

  • Format: 48-bit (6 bytes), 12 hexadecimal digits
  • Notation: 07:01:02:01:2C:4B (separated by colons)
  • Types: Unicast, Multicast, Broadcast
  • Commands: ipconfig /all, getmac

9. TCP vs UDP

TCP (Transmission Control Protocol)

Feature Description
Type Connection-oriented (three-way handshake)
Reliability Reliable with acknowledgment
Error Control Error checking with checksum
Flow Control Yes (sliding window)
Data Recovery Yes (sequencing)
Duplex Full duplex
Use Cases Web browsing, email, file transfer

TCP Header Components

  • Source Port, Destination Port
  • Sequence Number, Acknowledgment Number
  • Data Offset, Flags (SYN, ACK, FIN, RST, PSH, URG)
  • Window Size, Checksum, Urgent Pointer

Three-Way Handshake

Client ‚Üí Server: SYN (Sequence=X)
Server ‚Üí Client: SYN+ACK (Sequence=Y, Ack=X+1)
Client ‚Üí Server: ACK (Ack=Y+1)
Connection Established!

TCP Flow Control

  • Sliding Window: Allows multiple segments in transit
  • Window Size: Number of bytes that can be sent before acknowledgment
  • Fixed vs Sliding Window: Sliding allows dynamic adjustment

UDP (User Datagram Protocol)

Feature Description
Type Connectionless (no notification)
Reliability Unreliable, best-effort delivery
Error Control Minimal (checksum only)
Data Recovery None
Overhead Very low
Use Cases Video streaming, DNS, VoIP, gaming

UDP Header Components

  • Source Port, Destination Port
  • Length, Checksum

TCP vs UDP Comparison

Feature TCP UDP
Connection Connection-oriented Connectionless
Reliability High Low
Speed Slower Faster
Header Size 20 bytes minimum 8 bytes
Flow Control Yes No
Ordering Guaranteed Not guaranteed
Broadcast No Yes
Examples HTTP, SMTP, FTP DNS, DHCP, VoIP

10. Application Layer Protocols

Email Protocols

SMTP (Simple Mail Transfer Protocol)

  • Used by mail servers to send/receive messages
  • Port 25 (unencrypted), 587 (encrypted)

POP3 (Post Office Protocol v3)

  • Downloads messages to local device
  • Deletes from server (by default)
  • Works offline
  • Port 110

IMAP4 (Internet Message Access Protocol v4)

  • Retrieves message headers (not entire messages)
  • Keeps messages on server
  • Synchronizes across devices
  • Port 143

Email Client Types

  • Web-based: Gmail, Hotmail
  • Desktop: Microsoft Outlook, Thunderbird

Telnet

  • Remote access to computers
  • Insecure (sends data in plaintext)
  • Port 23
  • Alternatives: SSH (Port 22, encrypted)

11. Network Devices

NIC (Network Interface Card)

  • Also called: Network adapter, LAN adapter
  • Layer: Physical (Layer 1)
  • Functions: Interface between PC and network
  • Types: Internal (PCIe), External (USB, Wireless)
  • Address: MAC address burned on card

Repeater

  • Layer: Physical (Layer 1)
  • Function: Regenerates signal before it becomes weak/corrupted
  • Note: Does not amplify, only regenerates
  • Ports: 2
  • Limitation: Half-duplex communication

Hub

  • Layer: Physical (Layer 1)
  • Function: Multiport repeater
  • Behavior: Broadcasts data to all connected devices
  • Issues:
    • No data filtering
    • Single collision domain
    • CSMA/CD required
    • Network slow during congestion

Collision Domain

  • All devices connected to a hub share one collision domain
  • Collisions occur when two devices transmit simultaneously
  • Makes network slow and congested

CSMA/CD (Carrier Sense Multiple Access with Collision Detection)

  • Protocol used with hubs
  • Process:
    1. Listen before transmitting
    2. If medium idle, transmit
    3. If collision detected, send jam signal
    4. Backoff and retry

Switch

  • Layer: Data Link (Layer 2) - primarily
  • Function: Forwards traffic based on MAC address
  • Benefits:
    • Reduces traffic
    • High performance
    • Error checking before forwarding
    • Less collision domain (one per port)

Switch Types

Type Layer Function
Layer 1 Switch 1 Switching hub
Layer 2 Switch 2 LAN switch (MAC address forwarding)
Layer 3 Switch 3 Routing switch (IP address forwarding), Inter-VLAN routing

Router

  • Layer: Network (Layer 3)
  • Function: Routes data packets between different networks based on IP address
  • Components: CPU, Motherboard, RAM, ROM
  • Features:
    • Dynamically updating routing table
    • Protocol dependent
    • Connects LANs and WANs
    • Has network adapters with IP addresses

DHCP (Dynamic Host Configuration Protocol)

  • Automatically assigns IP configuration to devices
  • Assigned Information: Host IP, Subnet mask, Default Gateway, DNS server, Lease Time
  • Use Case: End-user devices
  • Static Addresses: Used for network devices (gateways, switches, servers, printers)

Splitter

  • Divides telephone signal into multiple signals
  • Each signal carries selected frequency range
  • Can reassemble signals from multiple sources

Home Router Functions

  • Main Function: Routing
  • Additional Functions:
    • Switch (multiple LAN ports)
    • DHCP server
    • Firewall
    • Access point (wireless)
