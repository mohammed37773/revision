1. INTRODUCTION TO TESTING ğŸ§ª

Definition:

Testing is the process of evaluating a system or its component(s) with the intent to find whether it satisfies the specified requirements or not. It is executing a system in order to identify any gaps, errors, or missing requirements in contrary to the actual requirements.

Core Principle:

"Testing always gives confidence in the developed software."

Testing Philosophy:

  • Think Like Developers â†’ Hunt It like Tester
  • Testing uncovers defects before delivery to customer

Level 1: Unit Testing ğŸ¯

Definition: Testing individual units/components of software

Purpose: Validate that each unit performs as expected

Who Performs: Developers (during coding phase)

Characteristics:

  • Isolates code section for verification
  • Done during development
  • Can be manual or automated

Benefits:

  • âœ… Fix bugs early in development cycle
  • âœ… Saves costs
  • âœ… Helps understand codebase
  • âœ… Serves as project documentation
  • âœ… Enables code re-use

Level 2: Integration Testing ğŸ”—

Definition: Testing software modules integrated logically as a group

Purpose: Expose defects in interaction between integrated modules

Focus: Data communication among modules

Key Questions:

  • Do modules work together correctly?
  • Is data passed correctly between modules?

Level 3: System Testing ğŸŒ

Definition: Testing complete and fully integrated software product

Purpose: Evaluate end-to-end system specifications

Characteristics:

  • Tests complete computer-based system
  • Software is tested with other hardware/software systems
  • Series of different tests to exercise full system

Level 4: Acceptance Testing âœ…

Definition: Testing performed by end-user or client before production

Types:

  • User Acceptance Testing (UAT)
  • Business- Contract Acceptance Testing Acceptance Testing
  • Regulation Acceptance Testing

Characteristics:

  • Done after functional, integration, and system testing
  • Validates end-to-end business flow
  • Uses production-like data
  • Black box testing with end-users
  • NOT focused on cosmetic errors or spelling
